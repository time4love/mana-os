# 🗺 Mana OS — מפת דרכים וארכיטקטורה

## החזון
Mana OS היא מערכת הפעלה מבוזרת ונטולת כסף לקהילות שמנהלות את עצמן. אנחנו פועלים אך ורק על **מאנה** (משאבים טבעיים + הון אנושי + פוטנציאל פעולה). מטבע פיאט, מחירים ומושגים כלכליים מסורתיים לא קיימים במערכת.

## 🏗 ארכיטקטורת מערכת
- **חוזים:** Solidity, Foundry. ביצוע ללא גז (ERC-4337).
- **פרונט:** Next.js 15 (App Router), TailwindCSS (תכונות לוגיות ל-RTL/עברית), shadcn/ui.
- **ממשק Web3:** Wagmi, Viem.
- **אורקל AI:** Vercel AI SDK + OpenAI/Anthropic (תכנון משאבים ב-JSON מובנה).
- **DB off-chain:** Supabase.

## 🔄 זרימה הוליסטית: מהרעיון להצעה לביצוע

מחזור החיים של הצעה ב-Mana OS מחבר את כל השלבים לזרימה אחת:

1. **אידאה** (שלב 4) — חבר קהילה מתאר פרויקט בטקסט חופשי. **האורקל הסוקרטי** (מורה רוחני וארכיטקט) מלווה: שואל בעדינות, מנחה לפתרונות aligned לטבע, ורק אז מפיק **ProposalResourcePlan** מובנה (משאבים טבעיים + הון אנושי) דרך הכלי `finalize_resource_plan`.
2. **ממשל** (שלב 5) — ההצעה (או ה-hash שלה) נרשמת on-chain ב-**ProposalsDAO.sol**. ממשק "המדורה" מאפשר לצפות בהצעות ולהצביע עם הארנק. סטטוס: **Pending Vote** → **Approved** / **Rejected**.
3. **הקצאת משאבים** (שלב 6) — **הדהוד מבוסס משיכה**: הצעות שאושרו מופיעות ב-**לוח הקריאות**. המשתמשים טוענים קריאות לפי תהודה; בלי התחייבות כפויה. סטטוס: **Open** → **In Resonance** → **Completed**.
4. **הצטרפות** (שלב 7) — **חוזי נשמה דינמיים**: חברים חדשים עוברים "הצטרף לקהילה", מכריזים על **יכולת עונתית** (מנוחה, בנייה, למידה). מנחה (Level 3) מאשר ומנפיק SBT ראשון (שוליה או בסיסי).
5. **מפת ההתעוררות** (שלב 8) — מצב סימולציה: שכבת מפה משנה ייעוד תשתיות; משתמשים מניעים זרעי חזון, סיכות שפע והתחייבויות משאבים.
6. **פריסה והשקה** (שלב 9) — חוזים ופרונט ל-testnet ו-production; הרפו נפתח לתורמים.

- [ ] **מיפוי המונים (Crowdsourcing Abundance):** פיתוח ממשק המאפשר למשתמשים לנעוץ 3 סוגי סיכות במרחב הפיזי:

  3. *פתיחת גבולות (Resource Pledging):* אזרחים שפותחים את השטח הפרטי שלהם לשימוש קהילתי (למשל גג משותף).
